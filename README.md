# Software Development Methodology and Theory Repository

一个聚焦于**软件开发方法论与 AI 协作理论**的知识仓库，用于沉淀概念框架、工作流分析、应用案例与研究资料，支持个人研究、团队共创与长期迭代。

## 项目定位

本仓库主要回答以下问题：

- AI 如何改变软件开发中的协作关系与角色分工？
- 自动化在研发流程中的边界、收益与风险是什么？
- 如何将理论落地到真实业务与工程场景中？

目标是形成一套可复用的认知体系，帮助团队在技术升级时兼顾效率、质量与治理。

## 各目录说明

- `01_concepts`：核心概念与理论框架，定义关键术语和分析模型。
- `02_workflow-analysis`：研发工作流视角下的 AI 集成、自动化拆解与敏捷实践。
- `03_use-case-studies`：分领域案例研究，关注可迁移经验与失败教训。
- `04_research-papers`：研究论文与学术视角提炼，连接实践与理论证据。
- `05_further-reading`：书单与精选文章，作为持续学习入口。

## 使用方式

- 作为研究索引：从 `01_concepts` 开始建立统一认知，再阅读后续模块。
- 作为团队共识文档：在方案评审或流程改造前后进行对照。
- 作为写作素材库：将案例、观点、引用来源沉淀到对应主题文件中。

## 写作建议

- 每篇文档建议包含：背景、核心观点、关键论据、实践启示、参考来源。
- 尽量区分“事实”“推断”“观点”，提升可追溯性与讨论质量。
- 对新增案例注明场景边界（团队规模、行业、阶段）以便复用。
